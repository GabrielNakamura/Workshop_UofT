The tools - Git, Github and RStudio

Gabriel Nakamura

Here you will find instructions on how to install git, create an account in github and connect both in RStudio.

Git e GitHub - downloading and installing

Git can be installed here.

You can create a GitHub account here[https://github.com/]

Git e Git client

We are gonna use as a a Git client RStudio, that can be downloaded from here. PS - You will find out that RStudio changed the name, now it is called Posit

Other Git client exist, but today we are going to use only RStudio.

Installing and configuring Git in your computer

First we need to know if Git is already installed in your machine. To check if it is already living in your computer you can type:

which git

If it appears a message in your computer indicating the path in which git is installed it means you already have git installed.

Connecting Git and GitHub using RStudio

For this task we will use some R libraries, so if you do not have these libraries you can install them by running the following code:

libs <- c("usethis", "gitcreds", "here")
if (!requireNamespace(libs, quietly = TRUE)){
    install.packages(libs)
  }

After instalation we need to configure git and connect it our GitHub account. We will make this using RStudio (our git client)

usethis::use_git_config(user.name = "YourUserName", user.email = "YorMail@example.br")

After informing your credentials we will need to provide a password. In this step we need to generate a token.

usethis::create_github_token()

This will make your default browser open and redirect you to GitHub webpage. We can do this mannualy by visiting Github settings and navigating to tokens, like in the image bellow

After generating the token we need to

library(gitcreds)
gitcreds::gitcreds_set()

You will see in the console a message asking to pass to it your token. After that you are set!!

Important - Save this token in a safe place, it works like your password. A suggestion is LastPass app

Git dictionary

Some terms used in version control workflow

  • directory = folder
  • commit = Commentaries on files
  • clone = Clone your remote repository in your machine
  • pull = baixar dados para (geralmente seu computador)
  • push = subir dados para (geralmente para um servidor remoto a.k.a nuvem)
  • main ou master = pasta remota
  • repository ou reppo = diretório remoto
  • watch = monitorar o repositório e cada mudança feita nele
  • master = repositório remoto principal
  • branch = repositório remoto, pode ser o principal ou repositórios adicionais
  • merge = mesclar dois arquivos
  • diff = observer a diferença entre dois arquivos
  • stage e unstaged = significa selecionar (stage) ou não (unstaged) um arquivo especifico para um commit
